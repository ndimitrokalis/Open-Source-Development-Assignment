# 🚀 Contributing to LibreHelp 

**LibreHelp** is an open-source customer support system designed to help organizations manage support tickets and knowledge base articles efficiently.

We welcome contributions that improve the functionality, stability, documentation, and usability of the project.

---

## 📌 Project Overview

LibreHelp is a lightweight helpdesk platform that allows organizations to manage customer support interactions through a ticketing system and knowledge base.

Main technologies used in this project include:

* Python
* Flask
* SQLAlchemy
* Bootstrap
* Jinja Templates
* MySQL or SQLite

---

## 📥 How to Contribute

### 1. Clone the Repository

Clone the project to your local machine:

git clone https://github.com/ndimitrokalis/Open-Source-Development-Assignment.git

Navigate to the project directory:

cd Open-Source-Development-Assignment

---

### 2. Create a Feature Branch

Always create a new branch for your work.

Example:

git checkout -b feature/ticket-system

Branch naming convention:

feature/feature-name
bugfix/issue-name
docs/documentation-update

---

### 3. Setup the Development Environment (Windows)

Create a virtual environment:

python -m venv venv

Activate the virtual environment:

venv\Scripts\activate

Install project dependencies:

pip install -r requirements.txt

---

### 4. Run the Application

Start the development server:

flask run

The application should now be running locally.

---

### 5. Making Changes

When contributing code, please:

* Follow standard Python coding practices
* Write clear and readable code
* Keep functions small and focused
* Add comments when necessary to clarify logic

---

### 6. Committing Changes

Use clear and descriptive commit messages.

Example:

Add ticket creation functionality

Try to keep commits focused on a single change or improvement.

---

### 7. Push and Create a Pull Request

Push your branch to GitHub:

git push origin feature/ticket-system

Then open a Pull Request and describe:

* What the change does
* Why it is needed
* Any related issues

---

## 🐞 Reporting Issues

If you encounter a bug, please open an issue including:

* A clear description of the problem
* Steps to reproduce the issue
* Expected behavior

---

## 💡 Feature Suggestions

Suggestions for improvements or new features are welcome.

When submitting a feature request, please include:

* The problem the feature solves
* A brief explanation of the proposed solution

---

## 📚 Documentation Contributions

Documentation improvements are highly appreciated.

You can contribute by:

* Improving the README file
* Expanding setup instructions
* Adding developer documentation
* Clarifying existing documentation

---

Thank you for helping improve **LibreHelp** and contributing to the project! 🎉
